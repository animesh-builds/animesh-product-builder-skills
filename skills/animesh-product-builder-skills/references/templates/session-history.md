# Session History

Running log of planning and build sessions. Newest at top.
On a new device, paste this at the start of a Claude conversation so context carries over.

**Update rules:**
- Always prepend new entries (newest at top).
- Keep entries concise — bullet points, not paragraphs.
- If a session spans a day boundary, one entry is fine; use the date you started.
- Never delete old entries. If something was wrong, add a correction note in the current entry.
- Don't duplicate full doc contents — just note what was touched.
- Decisions live in `decisions.md`; only reference IDs here.
